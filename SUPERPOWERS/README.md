# SUPERPOWERS WORKFLOW

Workflows profesionales adaptados de Superpowers para Kimi K2.5.

## PHASE 1: DISEÑO

| SKILL | DESCRIPCIÓN |
|-------|-------------|
| BRAINSTORMING-DESIGN | Planificación Socrática antes de codear |

## PHASE 2: DESARROLLO

| SKILL | DESCRIPCIÓN |
|-------|-------------|
| TDD-WORKFLOW | Red-Green-Refactor para lógica |
| PROJECT-EXECUTION | Ejecutar planes paso a paso |
| CODE-REVIEW | Revisión de calidad obligatoria |

## PHASE 3: CALIDAD

| SKILL | DESCRIPCIÓN |
|-------|-------------|
| SYSTEMATIC-DEBUGGING | Debug metódico sin adivinar |
| GIT-WORKFLOW | Control de versiones profesional |

## META

| SKILL | DESCRIPCIÓN |
|-------|-------------|
| WRITING-SKILLS | Crear nuevas skills correctamente |

---

## WORKFLOW COMPLETO

```
1. BRAINSTORMING-DESIGN → DESIGN.md aprobado
2. PROJECT-EXECUTION → Plan con tasks
3. Por cada task → CODE-REVIEW → GIT-WORKFLOW
4. Si error → SYSTEMATIC-DEBUGGING
```
